3.0.0  WORKFLOW DISCIPLINE

3.1.0  BRAND STRICT MODE (MANDATORY)
3.1.1  Read LBPF_BRAND_PACK.md before any CSS/HTML work
3.1.2  Use only defined tokens (colors, spacing, radii, shadows)
3.1.3  Propose new visual needs in brand pack FIRST
3.1.4  Get approval before adding to brand-components.css
3.1.5  Then use in page-specific CSS

3.2.0  COMPONENT DISCIPLINE
3.2.1  Maintain modular architecture (separate JS/CSS files)
3.2.2  Reuse existing components before creating new ones
3.2.3  Keep components template-agnostic where possible
3.2.4  Document component dependencies

3.3.0  FILE ORGANIZATION
3.3.1  HTML files in root (index.html, lxtourguide.html, etc.)
3.3.2  CSS in css/ folder (template.css, brand-components.css, page-specific)
3.3.3  JS in js/ folder (components/, pages/, app.js)
3.3.4  Data in data/ folder (tours.js, guides.js)
3.3.5  Documentation in docs/ folder (primes/, core/, archive/)

4.0.0  SAFETY RULES

4.1.0  BEFORE MAJOR CHANGES
4.1.1  Create dated backup
4.1.2  Verify git status is clean
4.1.3  Get explicit user approval
4.1.4  Document reasoning in commit message

4.2.0  WHEN EDITING FILES
4.2.1  Read file first to understand context
4.2.2  Preserve existing indentation and style
4.2.3  Test changes before committing
4.2.4  Verify no broken links or references

4.3.0  WHEN MOVING CONTENT
4.3.1  Copy to destination first
4.3.2  Verify content exists in new location
4.3.3  Then delete from source
4.3.4  Update all references
